# Tech

[[frontend]]

RegisterPage.jsx: Страница регистрации. Должна показывать сообщение: "Проверьте почту для подтверждения", если бэкенд вернул 200 OK.

    ConfirmEmailPage.jsx: Сюда пользователь попадает из письма.

        Логика: Берет token из URL и делает GET запрос на /users/confirm.

    LoginPage.jsx: Форма входа. После успеха должна перенаправлять в профиль.

    ProfilePage.jsx: Защищенная страница. Делает запрос на /users/profile. Если бэкенд выдает 401, редиректим на логин.

2. Компоненты (src/components/)

    AuthContext.jsx (Критически важно): Контекст для хранения состояния пользователя (isLoggedIn, userData). Чтобы не переспрашивать сервер на каждой странице, залогинен ли юзер.

    ProtectedRoute.jsx: Обертка, которая не пустит неавторизованного пользователя в профиль.

3. Сервисы (src/api/)

    api.js: Настройка axios или fetch с параметром withCredentials: true. Твой бэкенд использует куки (response.set_cookie), поэтому фронтенд обязан отправлять их в каждом запросе.
